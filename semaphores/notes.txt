1. If the "full" semaphore is 0, what will happen when you perform a sem_wait on the "full" semaphore?
Answer: If the "full" semaphore is 0 and a thread calls sem_wait on it, the thread will block. It will wait until the "full" semaphore value becomes greater than 0 (indicating that there are items in the buffer to consume). In other words, sem_wait will cause the consumer thread to pause and wait until at least one item is produced and placed in the buffer, which will increase the "full" semaphore count.

Why does this happen?: The "full" semaphore tracks the number of items in the buffer that are available to be consumed. When it's 0, it means that the buffer is empty, and the consumer has to wait until there's something to consume. When an item is produced, the "full" semaphore is incremented, allowing the consumer to proceed.

2. How does the "full" semaphore change to a positive integer?
Answer: The "full" semaphore becomes a positive integer when a producer places an item into the buffer. When a producer inserts an item, it calls sem_post on the "full" semaphore. This increments the "full" semaphore by 1, signaling that there is now an item in the buffer available for the consumer to consume.

Why is this important?: The "full" semaphore is a way to keep track of how many items are currently in the buffer. The producer increases the "full" semaphore to indicate that it has added an item, making it available for consumption by the consumer.

3. What is the consumer doing with the buffer array? Storing values or accessing values?
Answer: The consumer is accessing values from the buffer. Specifically, the consumer removes items from the buffer to process or consume them. It doesn’t store anything in the buffer; rather, it consumes data that the producer has placed there.

How does this work?: The consumer works with the buffer by reading (or removing) values from the "full" slot (which is managed with the "in" index) and then potentially modifying or using those values.

4. What index will you be working with in the consumer: in or out?
Answer: The consumer will be working with the out index.

Why?: The out index represents the next full slot to be consumed in the buffer. When the consumer is ready to consume an item, it accesses the buffer at the position indicated by the out index. After consuming the item, the out index is updated to the next position (typically using modulo arithmetic to wrap around if necessary).

5. When the consumer is done, which semaphore will need to have one added to it (i.e., "full" or "empty")?
Answer: When the consumer is done consuming an item, it needs to increment the "empty" semaphore.

Why?: The "empty" semaphore tracks how many empty slots are available in the buffer for the producer to insert items. When the consumer removes an item from the buffer, it frees up space for the producer to insert new items. Therefore, the "empty" semaphore should be incremented (sem_post) to reflect that there is now one more empty slot available in the buffer.

What happens to "full"?: The "full" semaphore will be decremented by the consumer (sem_wait) when it consumes an item, indicating that the number of items available for consumption has decreased by one.

Summary of Key Concepts:
sem_wait: Blocks if the semaphore value is 0 (no resources available), otherwise decrements the semaphore value.
sem_post: Increments the semaphore value, indicating that a resource has been produced or consumed.
full semaphore: Tracks how many items are available for consumption. It is incremented when a producer adds an item to the buffer and decremented when a consumer consumes an item.
empty semaphore: Tracks how many empty slots are available in the buffer. It is incremented when a consumer consumes an item and decremented when a producer adds an item.
Indices:
Producer works with the in index (where it places items).
Consumer works with the out index (where it removes items).
